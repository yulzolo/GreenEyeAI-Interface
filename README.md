# GreenEye-Interface (2 группа)

## Цель команды
Разработать интеллектуальный модуль анализа состояния растений и пользовательский интерфейс для мониторинга и управления гидропонной системой, которая:
- анализирует изображения растений с камеры с помощью модели компьютерного зрения;
- выявляет возможные заболевания или дефицит питательных веществ;
- отображает данные датчиков и состояние системы в удобном интерфейсе;
- позволяет пользователю удалённо контролировать параметры среды.

## Задачи команды

### Разработка AI-модуля
1. Найти и изучить готовые датасеты растений (латук).
2. Подготовить данные для обучения модели.
3. Настроить пайплайн обучения модели:
   - загрузка данных
   - предобработка изображений
   - обучение простой модели классификации
   - тестирование модели
4. Разработать модель компьютерного зрения для анализа состояния растений.
5. Настроить получение изображений с камеры.
6. Реализовать обработку изображений и выдачу результата (например: здоровое растение / признаки заболевания / дефицит элементов).
7. Подготовить формат передачи результатов анализа на сервер или в интерфейс.

### Разработка пользовательского интерфейса (Frontend)
1. Спроектировать структуру интерфейса системы мониторинга.
2. Создать макет интерфейса (в Figma), включающий:
   - видеопоток с камеры
   - отображение растения
   - графики параметров среды (pH, температура, влажность, освещение)
   - блок диагностики растения
   - элементы управления системой
3. Реализовать модульную структуру интерфейса, позволяющую:
   - добавлять новые полки
   - удалять существующие
   - масштабировать систему
4. Определить необходимые данные, которые должны передаваться с сервера в интерфейс.
5. Реализовать отображение данных датчиков в реальном времени.
6. Обеспечить визуализацию состояния растения и результатов работы нейросети.

1. Прототип модели компьютерного зрения для анализа состояния растений.
